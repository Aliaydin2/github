Önce github da yeni repository oluşturuyoruz, (bu repository bir klasör halindedir. İsmi de pc deki klasörün ismi olacaktır ) repository i seçtiğimizde yeşil renkli (code) pulldown unda https adresini kopyalıyoruz, vscode da komut sekmesini açıyoruz. github daki klasörü hangi klasörün içine koyacaksak ona geliyoruz. cd klasör-ismi cd.. komutları ile. Lokal olarak (state de) git dosya projesi oluşturmuş oluyoruz 1.komut: git init 2.komut: git clone kopyalanan-adres Github da yani repository de olan dosyaları, dosyalarda değişik yapıldıysa değişiklik yapılmış dosyaları ya da yeni oluşturulmuş ama eklenmemiş dosyaları görebiliyoruz. Ara komut: git status yeni create edeceğimiz dosyaları state 'ye ekleme; yeni dosya index.html olsun 3.komut: git add index.html git add * (.(nokta) ya da * (yıldız) koyarsak hepsini ekliyor) Snapshot i almak; State den commit’e aktardım. Artık github dan bir farkı var, yeni bir index.html dosyası commitli. 4.komut: add commit -m "girilecek mesaj içeriği" Repository e bağlanılan branch ismini verir (dal) Ara Komut: git branch Şimdi de değişiklikleri github a ekleme. uzak 5.komut: git push -u origin main -------------------------------------------------------------- log ve commitler (Her yeni commit log içerisinde en üste gelir) Commit edilmiş tüm değişiklikleri görmemizi sağlar. Her commitin uzun bir kodu var. bunu kullanmak istediğimizde ilk 7 karakteri kopyalamamız yeterli olacaktır Ara komut: git log Yeni yapılan değişiklikleri yani bir commit yapmadan önceki (son) commit ile göndermek istiyorsak 6.Komut: git commit --amend Bize mesajı değiştirip değiştirmeyeceğimizi soruyor. Mesajı değiştirmeden ya da değiştirip çıkmak istersek :q kullanarak çıkabiliriz. ESC yazılarak da olabilir. Son kommitin sadece mesaj kısmını değiştirmek istiyorsak 7.Komut: git commit --amend -m "main 2 | yeni mesaj commit edildi." Herhangi bir sıradaki commit bilgilerini görmek istiyorsak Ara komut: git log -n 1 Commit kaydındaki numarasının en az 7 hanesini kopyalayarak commit i silme işlemi yapabiliriz. 8.Komut: git revert enaz7haneliNumara NOT: silme işlemi revert ile silindiğine dair yeni bir commit oluşturulur otomtik olarak. Buraya mesajda girilebilir ama kendisi zaten giriyor.Silme işlemini belirten revert commit ini de silerek bi önceki commit in silinmemiş durumuna dönebiliriz. Silme işlemini belirten commitlerin log kayıtlarından silinmesini istiyorsak 9.Komut: git reset --hard enaz7haneliNumara İki commit arasında herhangi bir dosyada yapılmış değişiklikleri görmek için ; 10.Komut: git diff 04c80af22bed0d9..bbf4f180e733f79 komutlar.html Dosya adı belirtmezsek tüm değişiklikleri gösterir. git diff 04c80af22bed0d9..bbf4f180e733f79 ----------------------------------------------------------- Branch 1.Komut: git branch Projede var olan tüm brancleri gösterir.Hangisinde olduğumuzu da. 2.Komut: git branch header Header isminde yeni bir branch oluşturdu. Ama yine olduğumuz branchtayız. 3.Komut: git checkout header Bu komut ile header branch ına geçmiş oluruz. 4.Komut: git checkout -b footer footer isminde yeni bir branch oluşturum, o branch içerisine girdik. NOT:Ayrıca yeni branch i oluştururken hangi branchte isek yeni branch onun altında oluşur ve o brenchin o anki özelliklerini ile yeni branch oluşmuş olur. Bu yeni branch de oluşturacağımız dosyalar diğer branclerde gözükmez. 5.Komut: git branch -D footer ismini belirttiğimiz branch ı silmemizi sağlar ----------------------------------------------------------- stash (NE KAYIT NE DE COMMİTDİR. GEÇİCİ OLARAK SAKLANIR) Her yeni stash en üste gelir stash in işlevi: son yapmış olduğumuz commit den itibaren yapmış olduğumuz değişiklikleri bir köşede saklayıp ihtiyacımız olduğunda bize getirmek. 1.Komut: git stash dosyamızın o anki halini stash içerisinde saklar 2.Komut: git stash list stash içerisinde sakladığımız dosyaları liste halinde görmemizi sağlar 3.Komut: git stash clear tüm stash listesini siler 4.Komut: git stash pop En üstteki kaydı bize geri getirir ve stash list inden getirdiği stash i siler. Değişiklikler yapılmışsa onları gösterir ve istersen add yap gibi bilgi verir. git * git commit -m "" hem ekleyip hem commit edebiliriz.(normal komutlarımızla) 5.Komut: git stash apply stash@{0} index nosuna göre stash bilgilerini getirir ama pop gibi stash listeden silmez. ----------------------------------------------------------- merge main de olduğumuzu düşünelim; (header ı main e ekliyoruz) 1.Komut: git merge header iki branch in ortak commitinden (header branch ının oluşturulduğu en baştaki) en sondaki commitlerine kadar hepsini bir araya getirip log a ekliyor main de olduğumuzu düşünelim; (footer ı main e ekliyoruz.) 2.Komut: git merge --squesh footer Bu işlemde bizden birleştirmenin olduğuna dair mesajlı bir commit yapmamızı istiyor. Main de log kayıtlarına (git log) baktığımızda main in commitleri ve birleştirilmeye dair tek commiti gösterir. Bu tek commit footer ın commitlerini görmemizi sağlıyor (denemedim) main de olduğumuzu düşünelim; (main e footer bilgilerini aktaracağız) 3.Komut: git rebase footer Bu işlemde log kayıtları ve değişiklikler merge deki gibi gelir ANCAK merge (birleştirme yapılmaz, bilgi aktarılmış olur) 4.Komut: git merge --abort conflict ile karşılaştığımız durumda merge işlemini geri almak için kullanılır. ----------------------------------------------------------- Conflict; Problem; İki arkadaş bir dosyada çalıştığında, pull ya da fetch yapıldığında bilgiler çakışır ve hangisinin doğru olduğu kontrol edilmesi gerekir.
